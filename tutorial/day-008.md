## 1）Robotics Stack Map：每一层到底在干嘛（建立直觉）

### A. 硬件层（Hardware）

**你可以把它理解成：机器人“身体”和“神经末梢”。ROS2 最终都要落到这里。**

1. **传感器（Sensors）**

* 相机：给你“像素”，适合识别/跟踪/定位（视觉里程计等）
* IMU：给你角速度/加速度，帮助姿态估计、短期稳定
* LiDAR：给你距离点云/激光扫描，适合建图/避障
* 编码器：给你轮子/关节转了多少（里程计 odometry 的核心之一）

2. **执行器（Actuators）**

* 电机/舵机/夹爪：真正产生动作的东西
* 关键点：执行器通常不直接听 ROS2 的“高层指令”，而是听“驱动/控制板”的低层指令（PWM、电流、速度环等）

3. **电机驱动 / 控制板 / MCU**

* MCU（microcontroller）就是“低层实时小脑”
* 它负责：读传感器（低层）、做控制环（比如 1kHz）、输出 PWM/电流、保护（过流/过温）
* 为什么需要它：主控（Linux）不一定够实时，控制环容易被系统抖动影响

4. **主控计算（x86/ARM/Jetson）**

* 运行 Ubuntu/ROS2/算法（感知/规划）
* Jetson 常见于需要 GPU（深度学习感知、合成数据）的场景

5. **电源/急停/通信总线**

* 电源：电池→稳压→各模块供电；功率/噪声/地线都会影响系统稳定
* 急停（E-stop）：安全底线，硬件级别能立刻切断输出
* 通信总线：

  * **CAN**：车规、可靠、抗干扰；底盘/电机常用
  * **UART**：串口；简单外设常用
  * **I2C/SPI**：板级传感器常用
  * **Ethernet**：高带宽，LiDAR/相机/主控通信常见

---

### B. 系统软件层（System Software）

**你可以把它理解成：机器人“操作系统 + 中间件 + 驱动适配层”。**

1. **Ubuntu / RT 内核（概念先知道）**

* 普通 Linux：更追求吞吐/公平调度；延迟可能抖动（jitter）
* RT（实时）内核（比如 PREEMPT_RT）：更追求“按时响应”，对控制回路更友好
  你今天只要记住一句：**RT 是为了让控制回路更稳定、更可预测。**

2. **ROS2 / DDS**

* ROS2 是机器人应用层框架（节点、topic/service/action、launch、参数、工具链）
* DDS 是 ROS2 常用的通信中间件（负责发现、传输、QoS 等）
* 你第一周学的 topic/service/action/QoS，本质上很多都“落在 DDS 这层去实现”。

3. **驱动节点（Drivers）**

* 作用：把硬件数据变成 ROS2 topic（或 service/action），把 ROS2 命令变成硬件可执行的指令
* 举例：

  * 相机驱动：发布 `/image`、`/camera_info`
  * LiDAR 驱动：发布 `/scan` 或 `/points`
  * 底盘驱动：订阅 `/cmd_vel`，发布 `/odom`、`/joint_states`

4. **tf2 / robot_state_publisher**

* **tf2**：管理各种坐标系之间的关系（你 Day6 已学）
* **robot_state_publisher**：把 **URDF 里定义的连杆/关节结构** + **joint_states（关节角/轮速）** 变成 tf2 里一整棵“机器人自身的 TF 树”
* 你要记住这个链路：

  * URDF 描述“结构”（骨架）
  * joint_states 提供“关节当前姿态”（动作）
  * robot_state_publisher 把二者合成 → 发布 TF（/tf, /tf_static）

---

### C. 机器人描述与可视化（Description & Visualization）

**你可以把它理解成：让人类和工具“看懂机器人长什么样、现在姿态如何”。**

1. **URDF / Xacro**

* URDF：机器人结构的 XML（link/joint，外形/惯量/碰撞）
* Xacro：URDF 的“宏语言”——解决重复、参数化（比如左右轮、多个关节）

2. **RViz**

* **可视化工具**：订阅 ROS2 topic（比如激光、点云、路径）、订阅 TF，然后画出来
* RViz 不是仿真：它不会替你算物理，不会让机器人“真的动”，它只是“展示你系统发布出来的东西”。

---

### D. 仿真层（Simulation）

**你可以把它理解成：没有真机也能跑，且能做自动回归测试。**

1. **Gazebo（物理仿真）**

* 提供世界、重力、碰撞、关节动力学、传感器模型
* 能通过桥接把仿真传感器/机器人状态“当成 ROS2 topic”发布出来

2. **Isaac Sim（高保真 + AI/合成数据）**

* 常见卖点：高质量渲染、GPU 加速、合成数据、与学习/训练结合更紧
* 今天不要求安装；你只要在图里把它放到“仿真层”，并标注“高保真/合成数据”即可

---

### E. 算法层（Algorithm）

**你可以把它理解成：机器人“大脑”。今天先认名字，别深挖。**

* Localization / Mapping：定位与建图（SLAM、AMCL…）
* Perception：感知（检测/分割/跟踪/点云处理）
* Planning：规划（全局路径 + 局部避障）
* Control：控制（PID、MPC、轨迹跟踪），输出给底盘/电机驱动

---

### F. 工程化层（你的强项）

**你可以把它理解成：让系统可复现、可评测、可回归、可上线。**

* rosbag / replay（你已会）：数据资产化、复现 bug、离线评测
* eval/benchmark：指标、基准集、对比实验（A/B in robotics）
* telemetry/debug：日志、trace、指标、rqt、系统监控
* CI/test/sim regression：每次改动跑仿真/回放/指标对比，避免“改一处坏一片”

## 6）可选加分动手：用 20~30 分钟把“URDF → TF → RViz”串起来

这一步能让你对“描述与可视化层”有非常直观的感觉（强烈建议做一次）。

### 6.1 安装必要包

```bash
sudo apt update
sudo apt install -y \
  ros-$ROS_DISTRO-xacro \
  ros-$ROS_DISTRO-robot-state-publisher \
  ros-$ROS_DISTRO-joint-state-publisher-gui \
  ros-$ROS_DISTRO-rviz2
```

### 6.2 新建一个最小 URDF（用 xacro 写也行）

```bash
mkdir -p week2/urdf_demo
nano week2/urdf_demo/simple_bot.urdf.xacro
```

粘贴下面内容：

```xml
<?xml version="1.0"?>
<robot name="simple_bot" xmlns:xacro="http://ros.org/wiki/xacro">
  <!-- base -->
  <link name="base_link">
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <box size="0.30 0.20 0.10"/>
      </geometry>
    </visual>
  </link>

  <!-- a fixed "lidar" link -->
  <joint name="lidar_joint" type="fixed">
    <parent link="base_link"/>
    <child link="lidar_link"/>
    <origin xyz="0.15 0 0.08" rpy="0 0 0"/>
  </joint>

  <link name="lidar_link">
    <visual>
      <geometry>
        <cylinder radius="0.04" length="0.05"/>
      </geometry>
    </visual>
  </link>

  <!-- a rotating "wheel" joint so joint_state_publisher_gui has something to move -->
  <joint name="wheel_joint" type="continuous">
    <parent link="base_link"/>
    <child link="wheel_link"/>
    <origin xyz="0 -0.12 0.03" rpy="0 1.5708 0"/>
    <axis xyz="0 0 1"/>
  </joint>

  <link name="wheel_link">
    <visual>
      <geometry>
        <cylinder radius="0.05" length="0.02"/>
      </geometry>
    </visual>
  </link>
</robot>
```

### 6.3 运行 robot_state_publisher（发布 TF）

**终端 A：**

```bash
source /opt/ros/$ROS_DISTRO/setup.bash
ros2 run robot_state_publisher robot_state_publisher \
  --ros-args -p robot_description:="$(xacro week2/urdf_demo/simple_bot.urdf.xacro)"
```

### 6.4 运行 joint_state_publisher_gui（给关节角度）

**终端 B：**

```bash
source /opt/ros/$ROS_DISTRO/setup.bash
ros2 run joint_state_publisher_gui joint_state_publisher_gui \
  --ros-args -p robot_description:="$(xacro week2/urdf_demo/simple_bot.urdf.xacro)"
```

你会看到一个 GUI 滑条，拖动 `wheel_joint`。

### 6.5 打开 RViz 看 TF 和机器人模型

**终端 C：**

```bash
source /opt/ros/$ROS_DISTRO/setup.bash
rviz2
```

在 RViz 里做：

1. 左下角 **Fixed Frame** 设置为 `base_link`
2. Add → **RobotModel**
3. Add → **TF**

你现在看到的就是：

* URDF 定义结构
* joint_state_publisher_gui 改关节状态
* robot_state_publisher 把结构+状态变成 TF
* RViz 订阅 TF + robot_description 可视化出来

如果你还想加一条“你第一周会的 tf2 CLI”：

```bash
ros2 run tf2_tools view_frames
# 会生成 frames.pdf（在当前目录）
```
